## 一、系统架构概述

SmartAgent3 采用 **LangGraph Supervisor-Agent** 架构，构建了一套完整的"分类 → 规划 → 执行 → 重规划 → 回复 → 记忆提取"六阶段智能对话流水线。系统通过 MCP（Model Context Protocol）协议动态接入外部工具服务，支持多领域跨域协作，并融合三层记忆系统和多人格引擎，实现有温度、有记忆的智能交互体验。

### 1.1 Supervisor 图执行流程

系统的核心执行流程由 LangGraph 状态图驱动，各节点按以下顺序串联执行：

| 节点 | 名称 | 功能描述 |
|------|------|---------|
| contextEnrich | 上下文增强 | 注入用户位置、时间、人格信息，检索相关记忆 |
| classify | 任务分类 | LLM 结构化输出，判断领域（domain）和复杂度（complexity） |
| planStep | 任务规划 | 将复杂任务拆解为多步骤执行计划，分配给不同 Agent |
| execute | 步骤执行 | 调用对应 Domain Agent 执行具体步骤，Agent 内部通过 ReAct 循环调用工具 |
| replan | 重规划评估 | 评估已完成步骤，决定继续执行下一步还是进入回复阶段 |
| respond | 回复生成 | 融合人格、记忆和情感标签，生成最终自然语言回复 |
| memoryExtract | 记忆提取 | 异步从本轮对话中提取新记忆，更新用户画像 |

对于 **simple** 复杂度的任务，系统跳过 planStep 直接进入 execute 节点；对于 **moderate** 和 **complex** 任务，则经过完整的规划和多步执行流程。

### 1.2 已接入的 MCP 服务与工具清单

当前系统共注册 **27 个工具**，来自 3 个 MCP 服务源：

| MCP 服务 | 传输方式 | 工具数 | 工具列表 |
|---------|---------|--------|---------|
| 高德地图 | SSE | 15 | `maps_geo`、`maps_regeocode`、`maps_text_search`、`maps_around_search`、`maps_search_detail`、`maps_direction_driving`、`maps_direction_walking`、`maps_direction_transit_integrated`、`maps_direction_bicycling`、`maps_weather`、`maps_ip_location`、`maps_distance`、`maps_assistant`、`maps_riding_direction`、`maps_future_weather` |
| 网易云音乐 | SSE | 8 | `search`、`get_song_detail`、`get_song_url`、`get_unblocked_url`、`get_lyric`、`get_playlist`、`get_album`、`get_artist` |
| 免费天气（内置） | 内置 | 4 | `free_weather_by_city`、`free_weather_by_coords`、`free_ip_location`、`free_geocode_city` |

### 1.3 Domain Agent 架构

系统包含 4 个专业领域 Agent，每个 Agent 拥有独立的工具集和系统提示词：

| Agent | 职责 | 可用工具数 | 说明 |
|-------|------|-----------|------|
| navigationAgent | 导航、地图、天气、定位 | 19 | 整合高德地图 15 工具 + 免费天气 4 工具 |
| multimediaAgent | 音乐搜索、歌词、歌单 | 8 | 接入网易云音乐 MCP 全部 8 个工具 |
| fileAgent | 文件搜索、打开、创建 | 11 | 文件系统操作（需文件系统 MCP 连接） |
| generalAgent | 通用问答、闲聊、汇总 | 0 | 纯 LLM 对话，无工具调用 |

## 二、测试用例分类体系

根据 Supervisor 的任务分类维度，测试用例按 **领域（domain）** 和 **复杂度（complexity）** 两个维度进行组织。

### 2.1 领域分类

| 领域 | 标识 | 触发条件 |
|------|------|---------|
| 导航地图 | `navigation` | 涉及天气、定位、路径规划、POI 搜索等 |
| 多媒体 | `multimedia` | 涉及音乐搜索、歌词、歌单、歌手信息等 |
| 文件系统 | `file_system` | 涉及文件搜索、打开、创建等 |
| 通用对话 | `general` | 闲聊、知识问答、建议咨询等 |
| 跨域复合 | `cross_domain` | 同时涉及多个领域的复合任务 |

### 2.2 复杂度分级

| 复杂度 | 标识 | 执行路径 | 典型特征 |
|--------|------|---------|---------|
| 简单 | `simple` | classify → execute → respond | 单步操作，1 个 Agent，0-1 次工具调用 |
| 中等 | `moderate` | classify → plan → execute → replan → respond | 多步串行，1 个 Agent，2-3 次工具调用 |
| 复杂 | `complex` | classify → plan → execute → replan（循环） → respond | 跨域协作，多 Agent，4+ 次工具调用 |

## 三、导航地图领域测试用例

### 3.1 天气查询（simple）

**测试目标**：验证免费天气工具的基本调用能力。

| 编号 | 测试输入 | 预期行为 | 涉及工具 |
|------|---------|---------|---------|

## 六、跨域复合任务测试用例（重点）

跨域复合任务是 SmartAgent3 Supervisor-Agent 架构的核心价值体现，系统需要将一个自然语言请求拆解为多个步骤，分发给不同 Domain Agent 协作完成。

### 6.1 双域协作（complex）

**测试目标**：验证两个领域 Agent 的协作能力。

| 编号 | 测试输入 | 预期拆解 | 涉及 Agent |
|------|---------|---------|-----------|
| CROSS-001 | "帮我搜索周深的《大鱼》歌词，同时查一下上海的天气" | Step 1: 搜索歌曲 → Step 2: 获取歌词 → Step 3: 查天气 → Step 4: 汇总 | multimediaAgent + navigationAgent + generalAgent |
| CROSS-002 | "帮我查一下北京的天气，然后推荐一首适合这种天气听的歌" | Step 1: 查天气 → Step 2: 根据天气搜索歌曲 → Step 3: 汇总 | navigationAgent + multimediaAgent + generalAgent |
| CROSS-003 | "帮我规划从上海到杭州的路线，再搜一首适合路上听的歌" | Step 1: 路径规划 → Step 2: 搜索歌曲 → Step 3: 汇总 | navigationAgent + multimediaAgent + generalAgent |

### 6.2 三域协作（complex）

**测试目标**：验证三个领域 Agent 的深度协作和复杂任务拆解能力。

| 编号 | 测试输入 | 预期拆解 | 涉及 Agent |
|------|---------|---------|-----------|
| CROSS-004 | "帮我查上海的天气，规划从上海虹桥到浦东机场的驾车路线，最后推荐一个浦东机场附近的餐厅" | Step 1: 查天气 → Step 2: 路径规划 → Step 3: POI 搜索 → Step 4: 汇总 | navigationAgent(×3) + generalAgent |
| CROSS-005 | "先定位我的城市，查天气，如果天气好就帮我搜一首适合出行的歌" | Step 1: IP 定位 → Step 2: 查天气 → Step 3: 搜索歌曲 → Step 4: 汇总 | navigationAgent + multimediaAgent + generalAgent |
| CROSS-006 | "帮我搜索周深的专辑列表，同时查找上海附近的 KTV，再告诉我上海今天的天气" | Step 1: 搜索专辑 → Step 2: POI 搜索 → Step 3: 查天气 → Step 4: 汇总 | multimediaAgent + navigationAgent + generalAgent |

### 6.3 链式依赖任务（complex）

**测试目标**：验证步骤间存在数据依赖的链式任务执行能力。

| 编号 | 测试输入 | 预期拆解 | 依赖关系 |
|------|---------|---------|---------|

| CROSS-008 | "帮我找苏州工业园区附近的充电桩，然后规划从我当前位置到最近充电桩的路线" | Step 1: 定位 → Step 2: 周边搜索充电桩 → Step 3: 路径规划到最近充电桩（依赖 Step 1 和 Step 2） → Step 4: 汇总 | Step 3 依赖 Step 1 和 Step 2 |
| CROSS-009 | "帮我查一下杭州的天气和温度，如果低于 10 度就推荐一首温暖的歌，如果高于 10 度就推荐一首欢快的歌" | Step 1: 查天气 → Step 2: 根据温度条件搜索歌曲（依赖 Step 1） → Step 3: 汇总 | Step 2 依赖 Step 1 的温度数据 |

## 八、记忆系统测试用例

SmartAgent3 实现了三层记忆架构：**工作记忆（内存）→ 记忆提取（LLM）→ 长期记忆（MySQL）**。

### 8.1 记忆提取与持久化

| 编号 | 测试步骤 | 预期行为 | 验证方式 |
|------|---------|---------|---------|
| MEM-001 | 发送"我叫张三，住在苏州" | 提取事实记忆：用户名称、所在城市 | 检查"智能记忆"面板出现新记忆条目 |
| MEM-002 | 发送"我喜欢听周深的歌" | 提取偏好记忆：音乐偏好 | 检查"智能记忆"面板出现偏好类型记忆 |
| MEM-003 | 发送"我不喜欢辣的食物" | 提取偏好记忆：饮食偏好 | 检查记忆面板 |

### 8.2 记忆召回与融入

| 编号 | 前置条件 | 测试输入 | 预期行为 |
|------|---------|---------|---------|
| MEM-004 | 已记录"喜欢黑色" | "帮我推荐一些穿搭" | 推荐内容应融入黑色偏好 |
| MEM-005 | 已记录"住在苏州" | "帮我查一下天气" | 应自动查询苏州天气，无需用户再次指定城市 |
| MEM-006 | 已记录"喜欢周深" | "帮我推荐几首歌" | 推荐应优先包含周深或类似风格的歌曲 |

### 8.3 记忆分类验证

系统支持 4 种记忆类型和 3 种记忆层级：

| 记忆类型 | 标识 | 示例 |
|---------|------|------|
| 事实 | `fact` | "用户住在苏州"、"从苏州到上海浦东机场约 148 公里" |
| 偏好 | `preference` | "用户喜欢黑色"、"用户喜欢听周深的歌" |
| 行为 | `behavior` | "用户经常查询苏州到上海的路线" |
| 情感 | `emotion` | "用户今天心情不好" |

| 记忆层级 | 标识 | 存储位置 | TTL |
|---------|------|---------|-----|
| 情景记忆 | `episodic` | 内存 | 30 分钟 |
| 语义记忆 | `semantic` | MySQL | 长期 |
| 人格记忆 | `persona` | MySQL | 长期 |

## 十一、端到端集成测试场景

以下为推荐的端到端完整测试流程，覆盖系统的核心能力链路。

### 场景 A：新用户完整体验流程

按以下顺序依次发送消息，验证系统的记忆积累和个性化能力：

1. **"你好"** → 验证基本问候和人格展示
2. **"我叫小明，我住在苏州"** → 验证事实记忆提取
3. **"我喜欢黑色和周深的歌"** → 验证偏好记忆提取
4. **"帮我查一下天气"** → 验证记忆召回（应自动查询苏州天气）
5. **"帮我搜索周深的《大鱼》歌词"** → 验证音乐搜索 + 歌词获取
6. **"帮我规划从苏州到上海浦东机场的驾车路线，顺便告诉我上海的天气"** → 验证跨域任务拆解
7. **"帮我推荐一些东西"** → 验证记忆融入（应结合黑色偏好和音乐偏好）

### 场景 B：复杂跨域任务压力测试

发送以下高复杂度消息，验证系统的极限拆解能力：

1. **"帮我搜索周深的《大鱼》歌词，查一下上海的天气，再规划从苏州到上海的驾车路线"** → 3 域协作，预期 4-5 步拆解
2. **"从南京到杭州，分别告诉我驾车、公交和骑行各需要多长时间"** → 单域多模式，预期 5+ 次工具调用
3. **"先定位我的城市，查天气，如果天气好就推荐一首欢快的歌，如果天气不好就推荐一首安静的歌"** → 条件判断 + 跨域协作

### 场景 C：人格切换对比测试

使用同一问题在不同人格下测试，验证人格系统的差异化表现：

1. 切换到**小智**，发送"帮我查一下北京的天气"
2. 切换到**贾维斯**，发送"帮我查一下北京的天气"
3. 切换到**阿尔弗雷德**，发送"帮我查一下北京的天气"

对比三次回复的语言风格、称呼方式和情感标签差异。

## 十二、已知限制与注意事项

| 限制项 | 说明 | 影响范围 |
|--------|------|---------|
| 文件系统 MCP 未连接 | `fileAgent` 的 11 个工具均不可用 | 文件搜索、打开、创建等功能无法使用 |
| 网易云音乐匿名模式 | 未配置 Cookie，部分高品质音频和个人歌单功能受限 | `get_song_url` 可能返回受限结果 |
| 高德地图 API 配额 | 免费 API Key 有每日调用次数限制 | 高频测试时可能触发限流 |
| LLM 响应延迟 | 复杂跨域任务涉及多次 LLM 调用，总耗时可能较长 | complex 任务通常需要 30-60 秒 |
| 浏览器 MCP 未连接 | 应用浏览器 MCP 服务未配置 | 网页浏览相关功能不可用 |
