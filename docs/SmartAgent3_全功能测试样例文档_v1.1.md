## 1. 概述

本文档为 SmartAgent3 项目提供了一套完整的端到端手动测试用例，覆盖从基础对话到高级功能（记忆、情感、人格切换、MCP 工具调用、复杂任务规划）的全部核心模块。请按照本文档的步骤和预期结果，在车机中控屏界面上进行测试。

**测试环境：**

- 前端界面：车机中控屏（Cockpit）
- 默认人格：小智（xiaozhi）
- 情感渲染：已启用（通过情感标签可视化）
- 记忆系统：已启用（底部"智能记忆"卡片，每 3 秒自动刷新）
- 人格切换：已启用（右侧助手面板标题旁的下拉框）

**系统架构说明（供测试参考）：**

SmartAgent3 采用 LangGraph Supervisor-Agent 架构，消息处理流程为：

> 用户输入 → ContextEnrich（上下文丰富）→ Classify（意图分类）→ [Plan（任务规划）| Execute（直接执行）] → Execute（步骤执行）→ Replan（重规划评估）→ Respond（生成回复）→ MemoryExtract（记忆提取）

其中，**复杂度（complexity）** 决定是否进入 Plan 节点：
- `simple`：直接进入 Execute，由单一 Agent 处理
- `moderate` / `complex`：先进入 Plan 节点，由 LLM 生成多步执行计划，再逐步执行

### 2.6 复杂任务规划测试（TC-PLAN 系列）

> **背景说明：** SmartAgent3 的 Supervisor 架构包含一个 **Plan-and-Execute 图**（`planNode` → `executeNode` → `replanNode`）。当任务被分类为 `moderate` 或 `complex` 复杂度时，系统会先调用 LLM 生成多步执行计划，再逐步调用各 Agent 执行，最后由 `generalAgent` 汇总结果。本节测试该规划能力的正确性与鲁棒性。

**测试前置条件：** 服务器日志可访问（`tail -f /tmp/sa3.log`），以验证 `[PlanNode]` 和 `[ClassifyNode]` 的输出。

#### 2.6.1 中等复杂度（moderate）规划测试

| 测试用例 # | 功能模块 | 测试步骤 | 预期结果 |
|-----------|---------|---------|---------|
| TC-PLAN-01 | 导航 + 信息查询（moderate） | 1. 输入"帮我查一下从当前位置到上海浦东机场的驾车时间，顺便告诉我现在上海的天气" | 1. 后端日志显示 `[ClassifyNode] complexity=moderate`<br>2. 日志显示 `[PlanNode] Plan generated: N steps`，步骤包含 `navigationAgent`（路线规划）和 `navigationAgent`（天气查询）<br>3. 助手最终回复包含驾车时间和天气信息两部分内容 |
| TC-PLAN-02 | 音乐 + 对话（moderate） | 1. 输入"播放一首周杰伦的歌，然后告诉我他最出名的三首歌" | 1. 日志显示 `[PlanNode]` 生成 2 步计划：第 1 步 `multimediaAgent`（搜索并播放音乐），第 2 步 `generalAgent`（回答问题）<br>2. 助手回复包含播放确认和歌曲推荐两部分 |

### 3.3 复杂度分类规则（供开发参考）

| 复杂度 | 触发条件 | 处理路径 |
|--------|---------|---------|
| `simple` | 单一领域、单步操作 | 直接进入 Execute，由单一 Agent 处理 |
| `moderate` | 同一领域的多步操作，或需要少量跨域协作 | 进入 Plan 节点，生成 2-3 步计划 |
| `complex` | 明确的跨域任务（`cross_domain`），或需要复杂条件判断 | 进入 Plan 节点，生成 3+ 步计划，可能使用 Plan-Execute 图 |
