## 一、系统架构与改造概述

SmartAgent4 在本次迭代中完成了记忆系统从"被动捕获"到"主动调度"的范式转变。系统通过引入**记忆技能化（Skills-based Memory）**理念，将记忆的定义权和执行权交还给 Agent，彻底解决了多轮任务上下文割裂、提取时机僵化以及模糊指代处理困难等问题。

### 1.1 核心改造点

1. **主动记忆工具**：新增 `memory_store`、`memory_search`、`memory_update`、`memory_forget` 四个内置工具，封装了底层的数据库 CRUD 操作。
2. **System Prompt 策略注入**：在 Agent 的系统提示词中注入了三大核心策略：
   - **任务总结（Store）**：多轮任务结束后主动总结并存储。
   - **模糊消解（Search）**：遇到模糊指代或需要隐式偏好时主动检索历史。
   - **状态更新（Update/Forget）**：用户指出错误或状态改变时主动修正。
3. **降级自动提取**：默认关闭了每轮对话后的自动 LLM 提取（受 `AUTO_EXTRACTION_ENABLED` 开关控制），大幅降低了 Token 消耗，同时保留了工作记忆的实时更新。
4. **后台服务兼容**：Agent 主动存入的记忆（`source: "agent_skill"`）完全兼容现有的记忆巩固（Consolidation）和意图预测（Proactive Engine）管道。

### 1.2 测试用例分类体系

本次测试样例集重点围绕记忆系统的技能化改造展开，分为以下几个核心维度：
- **主动存储与任务总结**：验证 Agent 在多轮任务结束时的总结与存储能力。
- **主动检索与模糊消解**：验证 Agent 应对模糊指代和跨时间线查询的能力。
- **状态更新与记忆修正**：验证 Agent 对用户状态变更的响应与记忆更新能力。
- **降级开关与工作记忆**：验证自动提取开关的行为及工作记忆的连续性。
- **异常与边界处理**：验证工具调用的参数校验和异常处理机制。

## 二、主动存储与任务总结测试用例

**测试目标**：验证 Agent 能否在完成复杂多轮任务后，根据 Prompt 策略主动调用 `memory_store` 工具，将核心决策和关键信息总结为一条完整的结构化记忆。

| 编号 | 测试场景 | 测试步骤与输入 | 预期行为与结果 | 涉及工具 |
|------|---------|--------------|--------------|---------|
| STORE-001 | 单轮事实记忆存储 | 用户："我叫李华，住在北京朝阳区。" | Agent 主动调用 `memory_store` 存入事实记忆（`type: "fact"`），回复确认已记住。 | `memory_store` |
| STORE-002 | 单轮偏好记忆存储 | 用户："我以后只喝无糖的咖啡。" | Agent 主动调用 `memory_store` 存入偏好记忆（`type: "preference"`），回复确认偏好。 | `memory_store` |
| STORE-003 | 多轮导航任务总结 | 1. 用户："我要回家"<br>2. Agent：询问家地址或直接规划<br>3. 用户："途径一下山姆超市"<br>4. Agent：规划包含途径点的路线 | 在最后一步回复时，Agent 必须主动调用 `memory_store`，将"回家路线途径山姆超市"作为一条完整的情景记忆（`kind: "episodic"`）存入。 | `maps_direction_*` → `memory_store` |
| STORE-004 | 多轮行程预订总结 | 1. 用户："帮我查下明天去上海的高铁"<br>2. Agent：提供车次<br>3. 用户："订上午10点那趟，顺便定个全季酒店"<br>4. Agent：确认预订信息 | 任务完成后，Agent 主动调用 `memory_store` 总结本次行程（包含车次、时间、酒店信息）并存储。 | `memory_store` |

**验证要点**：多轮任务的记忆不应被割裂为单句，而应由 Agent 在任务终点进行全局总结，确保记忆的完整性和上下文连贯性。

## 三、主动检索与模糊消解测试用例

**测试目标**：验证 Agent 在面对包含模糊指代或需要隐式偏好的指令时，能否主动调用 `memory_search` 检索历史记忆，并在回复中成功串联上下文。

| 编号 | 测试场景 | 测试步骤与输入 | 预期行为与结果 | 涉及工具 |
|------|---------|--------------|--------------|---------|
| SEARCH-001 | 模糊指代消解（路线） | 前置：已存入"昨晚规划了从公司到虹桥机场的路线"。<br>用户："把昨晚的导航路线再发我一遍。" | Agent 识别到模糊指代"昨晚的导航路线"，主动调用 `memory_search` 检索历史，成功获取路线信息并回复用户。 | `memory_search` |
| SEARCH-002 | 模糊指代消解（地点） | 前置：已存入"上周五在静安寺的某某餐厅吃饭"。<br>用户："上次那家餐厅叫什么来着？" | Agent 主动调用 `memory_search` 检索"上次餐厅"，准确回答餐厅名称。 | `memory_search` |
| SEARCH-003 | 隐式偏好应用 | 前置：已存入"用户喜欢吃微辣的川菜"。<br>用户："推荐几家附近的餐厅。" | Agent 在推荐前主动调用 `memory_search` 查询饮食偏好，推荐结果中优先包含微辣的川菜馆。 | `memory_search` → `maps_around_search` |
| SEARCH-004 | 无结果优雅降级 | 用户："我上个月买的那本书叫什么？"（数据库中无此记忆） | Agent 调用 `memory_search` 未找到结果，向用户坦诚表示没有相关记录，并询问是否需要补充信息。 | `memory_search` |

**验证要点**：Agent 必须在回复用户**之前**挂起当前操作，先调用 `memory_search` 获取必要信息，确保回复的准确性。

## 四、状态更新与记忆修正测试用例

**测试目标**：验证 Agent 在用户明确指出记忆有误或状态发生改变时，能否主动调用 `memory_update` 或 `memory_forget` 工具进行修正。

| 编号 | 测试场景 | 测试步骤与输入 | 预期行为与结果 | 涉及工具 |
|------|---------|--------------|--------------|---------|
| UPDATE-001 | 事实状态变更 | 前置：已存入"用户住在北京"。<br>用户："我上周搬家到上海浦东了。" | Agent 先调用 `memory_search` 找到旧地址的记忆 ID，然后调用 `memory_update` 将内容更新为"住在上海浦东"，并回复确认。 | `memory_search` → `memory_update` |
| UPDATE-002 | 偏好状态变更 | 前置：已存入"用户喜欢喝全糖奶茶"。<br>用户："我最近在减肥，以后奶茶都换成无糖的。" | Agent 检索到旧偏好记忆，调用 `memory_update` 修改偏好内容，并回复确认已更新。 | `memory_search` → `memory_update` |
| FORGET-001 | 明确要求遗忘 | 前置：已存入"用户前女友叫小红"。<br>用户："把关于小红的记忆都删掉，我不想再提了。" | Agent 检索相关记忆，调用 `memory_forget` 删除该条记录，并给予情感关怀的回复。 | `memory_search` → `memory_forget` |
| FORGET-002 | 记忆纠错 | 前置：已存入"用户不喜欢吃香菜"。<br>用户："你记错了，我其实挺喜欢吃香菜的。" | Agent 检索到错误记忆，调用 `memory_update` 修正（或 `memory_forget` 删除旧的再 `memory_store` 新的），并向用户致歉。 | `memory_search` → `memory_update` |

**验证要点**：更新或删除操作必须先通过 `memory_search` 获取目标记忆的准确 ID，确保操作的精确性。

## 六、异常与边界处理测试用例

**测试目标**：验证记忆技能工具的参数校验、边界值处理及异常捕获机制。

| 编号 | 测试场景 | 测试步骤与输入 | 预期行为与结果 |
|------|---------|--------------|--------------|
| EDGE-001 | 缺少必填参数 | 模拟 Agent 调用 `memory_store` 时遗漏 `content` 参数。 | 工具返回明确的错误提示："错误：content 不能为空..."，Agent 根据提示修正或向用户反馈。 |
| EDGE-002 | 枚举值越界 | 模拟 Agent 调用 `memory_store` 时传入非法的 `type`（如 `type: "unknown"`）。 | 工具返回错误提示："错误：type 必须是 fact/behavior/preference/emotion 之一..."。 |
| EDGE-003 | 数值范围裁剪 | 模拟 Agent 传入 `importance: 1.5` 和 `confidence: -0.5`。 | 工具内部自动将数值裁剪到 `[0, 1]` 范围内（即 1.0 和 0.0），并成功存入数据库。 |
| EDGE-004 | 更新空字段 | 模拟 Agent 调用 `memory_update` 时仅提供 `memoryId`，无任何更新字段。 | 工具返回错误提示："错误：至少需要提供一个要更新的字段..."。 |
| EDGE-005 | 数据库异常 | 模拟数据库连接断开时调用 `memory_search`。 | 工具捕获异常并返回："记忆检索失败：[具体错误信息]"，Agent 不会崩溃，而是向用户解释当前系统状态。 |

## 七、端到端综合测试场景

### 场景 A：从偏好建立到模糊调用的完整闭环

1. **建立偏好**：用户发送 "我平时出差只坐高铁，而且必须是靠窗的座位。"
   - *预期*：Agent 回复确认，并后台调用 `memory_store` 存入该偏好。
2. **日常闲聊**：用户发送 "今天天气真不错。"
   - *预期*：Agent 正常回复，不触发任何记忆存储（因无价值信息）。
3. **模糊调用**：用户发送 "帮我规划一下明天去杭州的行程，按老规矩来。"
   - *预期*：Agent 识别到"老规矩"，主动调用 `memory_search` 检索偏好，随后调用相关工具查询高铁班次，并在最终回复中强调已为您筛选靠窗座位。

### 场景 B：复杂状态变更与纠错

1. **初始状态**：用户发送 "我现在的车牌号是苏A·12345。"
   - *预期*：Agent 调用 `memory_store` 存入车牌信息。
2. **状态变更**：用户发送 "我换车了，新车牌是苏E·88888。"
   - *预期*：Agent 调用 `memory_search` 找到旧车牌记忆，调用 `memory_update` 更新为新车牌。
3. **验证更新**：用户发送 "查一下我的车今天限行吗？"
   - *预期*：Agent 调用 `memory_search` 获取最新车牌（苏E·88888），然后调用限行查询工具，返回正确结果。
